注意事项：
本工具只能处理docx格式word文档，如果是doc的，需打开后另存为docx，不能直接改名称；

目前不支持文档路径中带特殊字符，如空格等。

文档超过400页（最多）的话，生成很慢，尽量拆开来生成。

doc_M文件夹下为本软件需要的文档模板，其他格式模板可能导致生成错误。

需求、大纲、说明等文档可以不需要去除前后内容，对生成无影响；
生成追踪关系时，软件默认大纲中的每一个测试项表格都有相应用例表格与之对应
不能有复用的用例！！
不能有复用的用例！！
不能有复用的用例！！（重要事情说3遍）
需要复用的情况可以再最后生成完最终追踪后，删掉表格，添加复用某某用例即可。


功能说明：
1、下方输出框中，如果内容太多，可选择清空内容。
2、点击菜单中的帮助或者输出框下方的“显示帮助”按钮可显示帮助内容，帮助菜单中有版本信息；
3、使用本软件进行操作时，除表格自动填充、内容提取和章节号提取需要选择确定的文件，其余功能
    需选择文件夹，且文件夹中的文档名称应为xxx软件测评大纲、xxx软件测试需求、xxx软件测试说明（用例）
	xxx软件测试记录、xxx软件问题单、xxx软件第一轮回归测试说明、xxx软件第一轮回归测试记录等形式；
	不能有多个带有测试说明、测试记录、测试大纲等字眼的文档；
4、需要生成临时文档的功能，临时文档均生成在软件所在目录中；
5、可根据个人喜好在“编辑”菜单栏选择工具栏显示方式；
6、自动填充表格功能：
   @ 单元格标题为需要填充的单元格的前一单元格内容，如：想批量填充测试记录
   @ 文档中的测试时间，则单元格标题选择测试时间，填充内容填写具体时间即可；
   @ 如果文档中部分表格已经填好了时间，自动填充不会修改原有内容。
   @ 如果需要全部重新填写，需在填写完单元格标题后，选择“清空单元格”按钮。
   @ 如果需要添加内容时序号递增，则先勾选“序号递增”选框，在点击填充；应用场景为，初始的标识写错
   @ 需要批量修改等。
   @ 可批量填充内容，如文档中100个表格，前20个表格的测试时间填20190101，中间20个表格测试时间
   @ 填20190505，其余填20191010.则在填充时，填充数量写20，程序会先填写20个表格，修改填充内容
   @ 后在点击填充，软件会顺序填充，不会冲掉原填充内容。最后一次填充可不填写填充数量，软件自动
   @ 将剩余表格填充完毕。
7、提取章节号用来生成追踪关系等，目前只支持7级标题以内的提取，需要注意，文档中的章节标题格式需统一；
8、提取表格内容功能为，自动提取文档中所有表格中单元格标题相同的内容。如需要提取测试用例名称、标识
   @ 可在单元格标题中填写“测试用例名称”，“标   识”，点击提取，则所有表格的用例名称和标识均提取到指定
   @ 文档中的一个表格内；
   @ 这里注意，单元格标题一定要与word文档中的一致，如用例文档中的用例标识，中间带了3个空格，填写单元格
   @ 标题时，同样需要带3个空格。建议直接拷贝word表格中的内容粘贴后，提取。
9、 本软件还集成了8114规则工具、C代码去空行工具、拷贝文件到指定目录工具，更多内容待后续更新。
